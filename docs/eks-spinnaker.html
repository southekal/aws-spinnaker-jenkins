Install Spinnaker using Amazon EKS (Distributed Installation)

  • We are going to be installing Spinnaker by using Kubernetes V2 provider for Amazon EKS (container management)
  • Refer documentation: Amazon EKS setup that goes through the process of creating a managing and managed account on AWS using Cloudformation.

    Note: there is a bug on creating the managing.yml. Refer to common issues below to fix the problem
  • Cloudformation will create a two-subnet VPC, IAM roles, instance profiles, and a Security Group for EKS control-plane communications and an EKS cluster.
  • Ensure your ~~/.hal/config file has the data entered from the values of the managing and managed account.
  • Once the managed and managing account are created, launch and configure the EKS worker nodes which are then joined to the Spinnaker EKS cluster via the aws-auth-cm.yaml file.
  • In a Distributed Installation of Spinnaker on Kubernetes, each of Spinnaker's microservices are deployed separately which is highly recommended for production usage. Refer to: distributed installation documentation

Common issues
